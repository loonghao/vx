## RuntimeRequest 解析统一与 `@` / `::` 语法修复

修复了 `runtime@version::executable` 规范格式的解析问题。之前 CLI 命令各自独立使用 `split_once('@')` 解析，导致 `msvc@14.19::cl` 这样的规范格式被错误解析（`::cl` 被吃进了版本字符串）。

### 根本原因
- CLI 层（cli.rs）先按 `@` 拆分 → tool="msvc", version="14.19::cl"
- where_cmd.rs 后按 `::` 拆分 → 但此时 tool 中已无 `::` → exe_override 丢失
- 最终以错误的版本 "14.19::cl" 查找，必然失败

### 修复方案
1. **`RuntimeRequest::parse()` 增强**：同时支持两种格式
   - 规范格式：`runtime@version::executable`（先检查 `::` 再处理 runtime 部分的 `@`）
   - 兼容格式：`runtime::executable@version`（保持原有行为）
2. **`Display` 实现更新**：输出规范格式 `runtime@version::exe` 而非 `runtime::exe@version`
3. **CLI 命令统一使用 `RuntimeRequest::parse()`**：消除 5 处独立 `split_once('@')` 调用
   - Which/Where 命令
   - Uninstall 命令
   - Versions 命令
   - Shell Launch 命令
   - install handler 的 `parse_tool_spec()`
   - common.rs 的 `parse_tool_version()`
4. **where_cmd::handle() 签名更新**：接受 `&RuntimeRequest` 而非分散的参数
5. **测试补充**：6 个新测试验证规范格式解析、两种格式一致性、roundtrip 行为

### 修改文件
- `crates/vx-resolver/src/runtime_request.rs` — 核心解析器修复
- `crates/vx-resolver/tests/runtime_request_tests.rs` — 新增规范格式测试
- `crates/vx-cli/src/cli.rs` — 4 个命令统一使用 RuntimeRequest::parse()
- `crates/vx-cli/src/commands/where_cmd.rs` — 接受 RuntimeRequest 参数
- `crates/vx-cli/src/commands/install/handler.rs` — parse_tool_spec 使用 RuntimeRequest
- `crates/vx-cli/src/commands/common.rs` — parse_tool_version 使用 RuntimeRequest
