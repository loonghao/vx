# 2026-03-19

## Provider 下载 URL 批量修复

### LLVM Provider
- **问题**: LLVM 20+ 版本的 Linux/macOS 资产文件命名格式变化
  - 旧格式 (< 20): `clang+llvm-{version}-{triple}.tar.xz`
  - 新格式 (>= 20): `LLVM-{version}-{OS}-{Arch}.tar.xz`
- **修复**: `crates/vx-providers/llvm/provider.star`
  - 添加版本检测 (`_major_version`, `_uses_new_format`)
  - 新旧两套平台映射

### hadolint Provider
- **问题**: 资产命名使用小写 `hadolint-linux-x86_64` 而非 `hadolint-Linux-x86_64`
- **修复**: 更新 `_OS_MAP` 使用小写的 linux/macos/windows

### ImageMagick Provider
- **问题**: AppImage 文件名包含动态 commit hash (`ImageMagick-3f2f629-gcc-x86_64.AppImage`)
- **修复**: 改用系统包管理器安装 (apt/brew/winget)

### pnpm Provider
- **问题**: 资产命名 `pnpm-linux-x64` 而非 `pnpm-linux-x64`
- **修复**: 更新平台映射为字符串 `"win-x64"` 而非 tuple

### sccache Provider
- **问题**: 资产文件有 `v` 前缀 `sccache-v0.14.0-...tar.gz`
- **修复**: 更新 asset 模板为 `sccache-v{version}-{triple}.{ext}`
- **别名冲突**: 移除 `compiler-cache` 别名（ccache 使用该别名）

### spack Provider
- **问题**: tag 有 `v` 前缀但资产文件名没有
- **修复**: `github_asset_url` 使用 `"v" + version` 作为 tag

### rez Provider
- **问题**: 无安装策略，仅有 `download_url` 返回 None
- **修复**: 添加 `system_install` 使用 uvx/pip 安装

### xmake Provider
- **问题**: 资产命名格式变化，使用 `xmake-bundle-v{version}.{os}.{arch}` 而非 `xmake-v{version}.{os}.{arch}.tar.gz`
- **修复**: 更新 `download_url` 使用 `xmake-bundle` 格式，Linux/macOS 为单二进制文件

### yq Provider
- **问题**:
  1. 使用 Go 风格的 os 命名 (`darwin` 而非 `macos`)
  2. `install_layout` 返回 `binary` 类型时格式不正确
- **修复**:
  1. 添加 `_OS_MAP` 和 `_ARCH_MAP` 转换
  2. 重写 `download_url` 和 `install_layout` 使用正确的格式

### actrun Provider
- **问题**: macOS 使用 `.pkg` 安装包格式，不支持直接下载
- **修复**: macOS 使用 Python wheel zip 文件替代 `.pkg`
