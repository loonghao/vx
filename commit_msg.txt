feat(vx-config): refactor types into modular directory structure

BREAKING CHANGE: types.rs split into types/ directory

## Changes

### Type Module Restructuring
- Split monolithic types.rs (1664 lines) into 19 focused modules
- Each configuration type now has its own file for better maintainability
- Preserved all public API exports via types/mod.rs

### New Module Structure (types/)
- config.rs: VxConfig root structure with helper methods
- project.rs: ProjectConfig
- tool.rs: ToolVersion, ToolConfig
- python.rs: PythonConfig, PythonDependencies
- env.rs: EnvConfig, SecretsConfig
- script.rs: ScriptConfig, ScriptDetails
- settings.rs: SettingsConfig, ExperimentalConfig
- hooks.rs: HooksConfig, HookCommand
- service.rs: ServiceConfig
- dependencies.rs: DependenciesConfig and related types
- ai.rs: AiConfig
- docs.rs: DocsConfig
- team.rs: TeamConfig, CodeOwnersConfig, ReviewConfig, etc.
- remote.rs: RemoteConfig, CodespacesConfig, GitpodConfig, etc.
- security.rs: SecurityConfig, SecurityAuditConfig, etc.
- test.rs: TestConfig, CoverageConfig, TestHooksConfig, etc.
- telemetry.rs: TelemetryConfig, BuildTrackingConfig, OtlpConfig
- container.rs: ContainerConfig, DockerfileConfig, etc.
- versioning.rs: VersioningConfig

### CLI Commands
- Added Security, Team, Remote, Test, Deps command handlers
- Fixed UI formatting methods
- Fixed type imports from vx_config

### Tests
- Added unit tests for new config modules

This refactoring improves code organization and makes future
extensions easier by allowing each feature area to evolve independently.
